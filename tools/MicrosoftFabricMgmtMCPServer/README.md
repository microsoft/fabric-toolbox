# MicrosoftFabricMgmt MCP Server

A [Model Context Protocol (MCP)](https://modelcontextprotocol.io/) server that wraps the
[MicrosoftFabricMgmt PowerShell module](../MicrosoftFabricMgmt/), enabling AI assistants
(Claude, GitHub Copilot, etc.) to manage Microsoft Fabric resources through natural language.

## Features

- **72 dedicated MCP tools** covering the most common Fabric resource operations
- **2 escape-hatch tools** (`invoke_fabric_ps`, `invoke_fabric_api_request`) for any operation not covered
- **Persistent PowerShell session** — module imported once, auth state preserved between tool calls
- **3 authentication methods** — User Principal (interactive), Service Principal, Managed Identity
- **Zero Python-side auth dependencies** — all auth delegated to `Az.Accounts` inside PowerShell
- Matches the patterns of the existing [DAXPerformanceTunerMCPServer](../DAXPerformanceTunerMCPServer/) and [SemanticModelMCPServer](../SemanticModelMCPServer/)

## Quick Start

### 1. Build the PowerShell module (if not already built)

```powershell
cd ../MicrosoftFabricMgmt
.\build.ps1 -Tasks build
```

### 2. Run setup

```batch
cd tools\MicrosoftFabricMgmtMCPServer
setup.bat
```

This will:
- Create a Python virtual environment (`.venv`)
- Install Python dependencies (`fastmcp`)
- Install PowerShell dependencies (`Az.Accounts`, `PSFramework`, `MicrosoftPowerBIMgmt`)
- Generate `.vscode/mcp.json`

### 3. Configure your MCP client

#### VS Code (auto-generated by setup.bat)

`.vscode/mcp.json` is generated automatically. Restart VS Code to pick it up.

#### Claude Desktop

Add to `%APPDATA%\Claude\claude_desktop_config.json`:

```json
{
  "mcpServers": {
    "fabric-mgmt": {
      "command": "C:\\path\\to\\MicrosoftFabricMgmtMCPServer\\.venv\\Scripts\\python.exe",
      "args": ["C:\\path\\to\\MicrosoftFabricMgmtMCPServer\\server.py"]
    }
  }
}
```

Replace `C:\\path\\to\\` with the absolute path to this directory.

### 4. Start using it

In your AI assistant, the typical workflow is:

```
1. connect_to_fabric(tenant_id="<your-azure-tenant-guid>")
2. list_workspaces()
3. list_lakehouses(workspace_id="<guid-from-step-2>")
4. create_lakehouse(workspace_id="<guid>", display_name="My New Lakehouse")
```

## Authentication

### User Principal (interactive browser login)

```
connect_to_fabric(tenant_id="<tenant-guid>")
```

Opens a browser window for interactive Azure AD login. Requires the user to be at a
terminal that can launch a browser.

### Service Principal (non-interactive)

```
connect_to_fabric(
    tenant_id="<tenant-guid>",
    app_id="<app-registration-guid>",
    app_secret="<client-secret>"
)
```

Recommended for automation, CI/CD, and unattended use.

### Managed Identity

```
connect_to_fabric(use_managed_identity=True)
```

For workloads running on Azure VMs, App Services, or Azure Functions with managed identity enabled.

## Architecture

```
server.py          FastMCP server, registers all tools
core/
  powershell_session.py   Persistent pwsh subprocess manager (sentinel protocol)
tools/
  auth_tools.py           connect_to_fabric, get_auth_status, disconnect
  workspace_tools.py      Workspace CRUD + role assignments
  lakehouse_tools.py      Lakehouse CRUD + tables
  warehouse_tools.py      Warehouse CRUD + connection string
  notebook_tools.py       Notebook CRUD
  pipeline_tools.py       Data pipeline CRUD
  environment_tools.py    Environment CRUD + publish + spark compute
  semantic_model_tools.py Semantic model CRUD
  eventhouse_tools.py     Eventhouse + KQL database CRUD
  spark_tools.py          Spark pools + workspace settings
  admin_tools.py          Admin-level resource views
  capacity_tools.py       Capacities + domain management
  sql_database_tools.py   SQL database CRUD + connection string
  escape_hatch_tools.py   invoke_fabric_ps + invoke_fabric_api_request
```

## Troubleshooting

**Module not found at startup**
- Build the PowerShell module: `cd ../MicrosoftFabricMgmt && .\build.ps1 -Tasks build`
- Or set `FABRIC_MGMT_MODULE_PATH` to the module directory

**Interactive auth not working**
- Ensure the terminal running the server can open a browser
- Use Service Principal auth instead: `connect_to_fabric(tenant_id=..., app_id=..., app_secret=...)`

**Tool call times out**
- The default timeout is 120 seconds. Long-running operations (e.g. environment publish) may
  need more time — use `invoke_fabric_ps` with explicit `Get-FabricLongRunningOperation` polling

**PS errors not surfacing**
- All PS errors are caught and returned as JSON: `{"success": false, "error": "...", "error_type": "..."}`
- Check `get_auth_status()` to verify authentication is active
